**Scenario: Data Visibility by Role**

- **เจ้าหน้าที่พัสดุ (Procurement):** ต้องสามารถเห็นโครงการทั้งหมดของ **ทุกหน่วยงาน** ในระบบ
- **เจ้าหน้าที่ภายนอก (External/Dept User):** ต้องสามารถเห็นเฉพาะโครงการที่สังกัด **หน่วยงาน (Department) ของตนเอง** เท่านั้น

**Scenario: Default Data View**

- เมื่อเข้าหน้าจอครั้งแรก ระบบต้อง Default แสดงผลโครงการที่มีความเคลื่อนไหวภายใน **6 เดือนล่าสุด**

**Scenario: Status Summary Cards**

ระบบต้องแสดงกล่องสรุปสถานะ (Summary Cards) แยกตามบทบาทผู้ใช้งาน ดังนี้:

- **กล่องสำหรับเจ้าหน้าที่พัสดุ:** โครงการทั้งหมด, ยังไม่ได้มอบหมาย, รอการตอบรับ, กำลังดำเนินการ, เสร็จสิ้น, ยกเลิก, ด่วน, ด่วนที่สุด, ด่วนพิเศษ
- **กล่องสำหรับเจ้าหน้าที่ภายนอก:** โครงการทั้งหมด, ยังไม่เริ่ม, กำลังดำเนินการ, เสร็จสิ้น, ยกเลิก, ด่วน, ด่วนที่สุด, ด่วนพิเศษ

**Scenario: Quick Search**

- สามารถค้นหาแบบ Global จาก: เลขที่ลงรับ, โครงการ, ผู้รับผิดชอบ

**Scenario: Advanced Filter**

- `search` (String): Global text search (should search across `receive_no`, `title`, etc.)
- `title` (String): Partial match specifically on the project title.
- `dateFrom` / `dateTo` (ISO Datetime): Date range filter (typically filtering by `created_at`).
- `fiscalYear` (String/Number): Exact match.
- `procurementType` (Array of Strings): Exact match using `IN()`.
- `status` (Array of Strings): Exact match using `IN()`.
- `urgentStatus` (Array of Strings): Exact match using `IN()`.
- `assignees` (Array of UUIDs): Must check if the ID exists in *either* the procurement assignees OR contract assignees lists.
- `units` (Array of UUIDs)
- `myTasks` (Boolean): If true, automatically filter where the logged-in user's ID is in the assignees list.
- `sortBy` / `sortOrder` (String): Standard pagination sorting.
- Default  6 เดือนล่าสุด + เรียงตาม `receive_no` เอาอันล่าสุดขึ้นก่อน
- รองรับการกรองตาม: ชื่อโครงการ, ช่วงเวลา (Start-End), ปีงบประมาณ, ผู้รับผิดชอบ, ประเภทงาน, สถานะโครงการ และความเร่งด่วน
- **หมายเหตุ:** เจ้าหน้าที่ภายนอกจะ **ไม่เห็น** ตัวเลือกตัวกรอง "หน่วยงาน"
- Filter สถานะโครงการสำหรับพัสดุ:
    - ยังไม่ได้มอบหมาย
    - รอการตอบรับ
    - กำลังดำเนินการ
    - รออนุมัติยกเลิก
    - ยกเลิก
    - เสร็จสิ้น
    - การเงินส่งคืนแก้ไข
- Filter สถานะโครงการสำหรับคนนอก**:**
    - ยังไม่เริ่ม (ยังไม่ได้มอบหมาย + รอการตอบรับ)
    - กำลังดำเนินการ (กำลังดำเนินการ + รออนุมัติยกเลิก + การเงินส่งคืนแก้ไข)
    - ยกเลิก
    - เสร็จสิ้น

**Scenario: Filter Actions**

**Scenario: Dynamic Name Display**

- ชื่อผู้รับผิดชอบที่แสดงในตารางหลัก ต้องเปลี่ยนไปตามขั้นตอนปัจจุบัน (เช่น อยู่ในขั้นจัดซื้อ โชว์ชื่อทีมจัดซื้อ / อยู่ในขั้นบริหารสัญญา โชว์ชื่อทีมบริหารสัญญา)

**Scenario: Authorized Assignee Addition** 

- ผู้ที่มีสิทธิ์เพิ่มผู้รับผิดชอบได้แก่: **เจ้าหน้าที่พัสดุผู้รับผิดชอบเดิม, หัวหน้ากลุ่มงานของโครงการนั้น** และ **Admin**
- **ข้อจำกัด:** เจ้าหน้าที่ที่ถูกเพิ่มมา **ต้องอยู่ในกลุ่มงานเดียวกัน** กับโครงการนั้นเท่านั้น
- เพิ่มผู้รับผิดชอบได้สูงสุด **2 คน ต่อ 1 ขั้นตอน** (จัดซื้อ 2 คน / บริหารสัญญา 2 คน)
- **เพิ่มคนได้เมื่อ:** เมื่อโครงการอยู่ในสถานะ **กำลังดำเนินการ**

**Scenario: Project Return** 

- ผู้ที่มีสิทธิ์คืนงานได้แก่: **เจ้าหน้าที่พัสดุผู้รับผิดชอบเดิม**
- ระบบจะอนุญาตให้คืนงานได้ภายใต้เงื่อนไข **ต้องผ่านทุกข้อ** ดังนี้:
    1. **Project Step**: โครงการต้องอยู่ใน **ขั้นตอนที่ 1 (ยังไม่เริ่ม)** เท่านั้น
    2. **No Data Entry**: ต้องยังไม่มีการบันทึกข้อมูลใดๆ ลงในฟอร์มของโครงการนั้น
    3. **No Document Upload**: ต้องยังไม่มีการอัปโหลดไฟล์เอกสารใดๆ เข้าสู่โครงการ
- หากโครงการมีการขยับไปขั้นตอนอื่น หรือมีการบันทึกข้อมูล/ไฟล์แล้ว ระบบต้อง **ซ่อน (Hide)** ปุ่มคืนงานทันที
- เมื่อกดคืนงาน: โครงการต้องกลับไปที่ **Pool งานกลาง**, สถานะเปลี่ยนเป็น **"ยังไม่ได้มอบหมาย"**, และล้างชื่อผู้รับผิดชอบเดิมออก

**Scenario: Cancel Project**

- ระบบต้อง **ไม่อนุญาต (Disable)** ให้ทำการยกเลิกโครงการ หากโครงการอยู่ในสถานะดังนี้: **ปิดโครงการ** , **ยกเลิก, รออนุมัติยกเลิก** หรือ **การเงินส่งคืนแก้ไข**
- ในการกดยกเลิกโครงการทุกครั้ง ระบบต้อง **บังคับ (Required)** ให้ผู้ใช้กรอก "เหตุผลการยกเลิก" เสมอ หากไม่กรอกจะไม่สามารถดำเนินการต่อได้
- **เจ้าหน้าที่พัสดุ (Staff):** เมื่อกดยกเลิก จะเป็นการส่ง **"คำขอยกเลิกโครงการ"** เท่านั้น สถานะโครงการจะเปลี่ยนเป็น "รออนุมัติยกเลิก" และต้องได้รับการอนุมัติจากหัวหน้ากลุ่มงานก่อนจึงจะถือว่ายกเลิกสำเร็จ
- **หัวหน้ากลุ่มงาน (Workgroup Head):** มีสิทธิ์ในการกด **"ยกเลิกโครงการ"** ได้ทันทีโดยไม่ต้องผ่านการอนุมัติซ้ำ

**Scenario: Notification**

- **เมื่อมีการเพิ่มผู้รับผิดชอบ:** ส่งการแจ้งเตือนไปยัง **เจ้าหน้าที่ที่ถูกเพิ่มชื่อ** เข้าในโครงการนั้นทันที
- **เมื่อมีการคืนงาน:** ส่งการแจ้งเตือนไปยัง **หัวหน้ากลุ่มงานของโครงการนั้น** เพื่อให้ทราบว่ามีการคืนงานกลับสู่ Pool
- **เมื่อมีการขอยกเลิกโครงการ:** ส่งการแจ้งเตือนไปยัง **หัวหน้ากลุ่มงาน** เพื่อแจ้งให้ทราบว่ามีคำขอยกเลิกที่ต้องพิจารณาอนุมัติ
- **เมื่อคำขอยกเลิกถูกอนุมัติ/ปฏิเสธ:** ส่งการแจ้งเตือนกลับไปยัง **เจ้าหน้าที่ผู้ส่งคำขอ** เพื่อแจ้งผลการอนุมัติ/ปฏิเสธ

**Scenario: Log** 

- **ประวัติการจัดการผู้รับผิดชอบ:** บันทึกว่าใครเป็นคนเพิ่ม, เพิ่มเจ้าหน้าที่ท่านใดเข้าโครงการ และวันเวลาที่ทำรายการ
- **ประวัติการคืนงาน:** บันทึกว่าเจ้าหน้าที่ท่านใดเป็นผู้คืนงาน, คืนจากขั้นตอนใด และวันเวลาที่ทำรายการ
- **ประวัติการยกเลิกโครงการ:**
    - บันทึกผู้ส่งคำขอยกเลิก และเหตุผลที่ระบุ
    - บันทึกผู้อนุมัติการยกเลิก (ในกรณีเจ้าหน้าที่เป็นคนขอ)
    - บันทึกวันเวลาที่โครงการเปลี่ยนสถานะเป็น "ยกเลิก" โดยสมบูรณ์
